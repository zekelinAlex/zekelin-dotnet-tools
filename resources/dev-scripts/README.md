# dev-scripts

> **READ THIS BEFORE TESTING ANY CHANGE TO THE DEVKIT.**
>
> Generated by the **Zekelin .NET Tools** VS Code extension. This folder is ignored by git.

## What is `Install-TargetNugets.ps1`?

It repacks every local DevKit project into the local NuGet feed so that downstream projects (consumers of these packages) pick up your latest source changes instead of resolving a stale cached or previously published NuGet version.

Concretely, in order, it:

1. Reads `packLocalVersion` from `../.vscode/zekelin-dotnet-tools.json`.
2. Resolves the local NuGet feed — reuses a source named `Local` / `local`, or any existing source whose path is `C:\NuGetLocal`, or registers a new `Local` source at `C:\NuGetLocal` if neither exists.
3. Runs `dotnet pack` on every `*.csproj` under `../src/Dataverse/` plus `../src/Sdk/TALXIS.DevKit.Build.Sdk.csproj` (if present) with `-p:PackageVersion=<version>` into `../artifacts/`.
4. Copies every resulting `*.nupkg` into the resolved local feed directory.
5. Kills every running `dotnet.exe` process.
6. Clears the NuGet cache (`dotnet nuget locals all --clear`).

## When to run it — IMPORTANT

**Run this script before testing any change made under `src/Dataverse/` or `src/Sdk/`.**

Why: downstream projects (both inside this repo and elsewhere on the machine) resolve their `TALXIS.DevKit.*` packages from the local feed. If you edit source here and skip this script, downstream builds will silently keep using the **old** version, your change will appear to have no effect, and you will waste time chasing a phantom bug.

## Usage

From anywhere (the script locates the repo root from its own location):

```powershell
./dev-scripts/Install-TargetNugets.ps1
```

Or double-click from Explorer. Requires `dotnet` on `PATH` and PowerShell 5.1 or newer. No admin rights needed unless `C:\NuGetLocal` cannot be created by your user.

## Configuring the package version

The script reads the version from `../.vscode/zekelin-dotnet-tools.json`:

```json
{
  "packLocalVersion": "0.0.0.14"
}
```

To change the version, edit that file, or re-run **Generate Install Script** / **Install targets Nugets** from the Zekelin .NET Tools sidebar — they'll prompt for a new version when the file is missing.

## Regenerating this folder

Delete `dev-scripts/` and click **Generate Install Script** in the Zekelin .NET Tools sidebar again.
